# LiteShip Master Plan — the Effect Shed & Architecture Program

**380 files · provenance: 25-package audit (215 threads) → design research (12 agents) → 6 workstream planners → 4 core-seam mappers + synthesis. Every non-create path repo-verified. 2026-07-16.**

## Doctrine

Correctness is the baseline, not the goal. The program is governed by three laws:

1. **Never recompute what is already computed** (fusion / banana-split). Every migration below is a *transport swap* around a pure kernel that already exists — verified per seam. If a step requires rewriting domain logic, the step is wrong; stop and re-derive.
2. **Behavior is preserved byte-identically where bytes are law.** The receipt sha256:/fnv1a divergence, canonical CBOR, JSON-Schema output parity, and motion Law 4 (CSS/JS easing identity) are caged by committed fixtures before any producer changes.
3. **No temp hack becomes canon.** Nothing lands hedged. The static-QA gate (below) mechanically rejects the residue of compiler-appeasement.

## Methodology — AI-optimized careful big bang

Waves, not increments. Each wave lands whole or not at all — there is no "get it compiling and fix it later," because subagents make the careful path cheap:

1. **Type-first (TyDD):** the wave's contracts land as types before implementations — `SchemaPort`, law-table interfaces, tagged unions. Typed holes are `S.hole<A>` (loud, enumerable, decode-blocking) — never `undefined as never`, never `as unknown as`.
2. **Red (TDD):** tests are authored and RUN RED before implementation — law tables (Lifetime LIFO/exactly-once; CellKernel replay-1/no-replay; ECS within-tick read-current), property tests (fast-check, seeded), determinism pins (injected clocks — no Date.now in test paths), byte-parity fixtures captured from the CURRENT code before producers change.
3. **Green:** Opus implementer agents work isolated slices with disjoint file ownership; a single integration agent owns shared barrels (core index.ts) to prevent races.
4. **Static QA gate (mechanical):** banned-pattern sweep over the wave diff — `as unknown as`, `@ts-ignore`, `eslint-disable`, `.skip(`, `.only(`, `TODO|FIXME|HACK|XXX`, stray `console.log`, new `from 'effect'` imports in shed packages. Zero tolerance; a hit fails the wave.
5. **Adversarial QA (Fable):** a reviewer agent whose charter is to REFUTE the wave — hunt for behavior drift, weakened tests, fixture hand-editing, law-table gaps — before anything is called done.
6. **Gates:** `pnpm typecheck`, build, affected suites, gauntlet where enrolled. All green → one atomic commit per wave. Anything less → the wave does not land.

## Conflict resolutions (locked)

**spine-conformance.test.ts — DELETE, sequenced; extension rejected.** Both artifacts derive one fact — mirror ≡ source — by two folds over the same type surface: the pins hand-verify a subset per edit forever; gen-spine computes the whole mirror once per change. Banana-split says fuse: one traversal of the runtime type surface produces the pair (mirror bytes, proof), and the byte gate then checks the only remaining degree of freedom — staleness of the checked-in copy — across ALL exported types, not the pinned subset that already let CapSet's Set→array drift through (ROADMAP 125-128 names auto-derivation as the fix). The byte-compare subsumes the pins' assignability guarantee conditionally on generator fidelity; discharge that once by seeding gen-spine's conformance fixture with the existing pin pairs. Extending pins invests hand-recomputation in an artifact scheduled for deletion, and drags every future _spine retype (four seams this wave alone). Caveats: gen-spine does not exist yet; ADR-0010 brands flow spine→runtime, so brands are authored generator input.
*Preserved:* The pins are never-executed type-level code — no runtime behavior exists to lose. The runtime-existence tests (Config.make, Boundary, resolvePrimitive, dispatch) are kept: bytes cannot prove exports exist. The pinned subset's drift guarantee survives relocated into gen-spine's seed conformance fixtures plus the whole-file byte gate; pins stay frozen, not deleted, until that gate is green.

**SchemaPort — one permanent contract, retyped once, spine-first.** The two proposals are one fold run twice: retype-to-interim then retype-to-kernel traverses the identical slot structure (capsule input/output, codec schema, ecs PartShape.schema — and _spine 442/977-982, the mirror's only Schema refs) and discards the first result — textbook recomputation. Retyping to kernel values 'now' is not even executable: the kernel does not exist yet, so it degenerates into a placeholder anyway. Fusion: make the port BE the kernel's type contract. Effect v4 Schema/Codec values carry phantom Type/Encoded members, so SchemaPort<A,I> = { readonly Type: A; readonly Encoded: I } is structurally satisfied by every effect Schema today and by kernel schemas tomorrow — the kernel is built TO this contract, never retrofitted. The ordering that strands no work: SchemaPort lands in _spine §23 first (ADR-0010 mandates spine-before-implementation), core slots retype once this wave, and the schema wave later swaps only runtime decode machinery (TypeValidator, Codec.make internals, harness emissions) — the slots are never touched again.
*Preserved:* Type-level-only change, zero runtime delta: capsule authors keep passing effect Schema values (they satisfy the port structurally); TypeValidator/Codec/harness decode paths keep their effect imports until the schema wave; A/I inference through the slots is preserved by the phantom members; ecs PartShape.schema was already never decoded at runtime. Verify beta-32's Type/Encoded members before landing.
